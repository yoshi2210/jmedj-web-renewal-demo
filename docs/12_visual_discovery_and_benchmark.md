# v4: ビジュアル探索(サムネで探す)と、現代サービスとの比較評価

## 1. ユーザー指摘

1.「表紙(サムネ)で全体的に探せるUI/UXが必要では」
2.「今の検索・表示UIは、現代的なモダンデザインの同種の高度なサービスと比べてどうなのか」

## 2. 正直な比較評価(v3時点)

対象比較サービス: Amazon / 楽天ブックス / honto / 医書.jp(医学書EC)、Apple Books、YouTube、Netflix、Pinterest(ビジュアル探索)、note、Algolia InstantSearch、UpToDate・PubMed(医療情報)。

| 観点 | v3の状態 | 現代トップティア | 判定 |
|---|---|---|---|
| ビジュアル探索(サムネ/書影で一覧・想起) | テキストカードのみ。画像ゼロ | 一覧の主役は必ず表紙/サムネ | **決定的に不足** |
| 検索の即時性(ライブ・サジェスト・件数) | 実装済み | 同等 | 及第 |
| 検索結果のファセット絞り込み | 一覧ページのみ。検索結果画面には無い | 検索結果にファセット常設 | 部分不足 |
| 情報設計・状態設計・a11y | IA整理・空/読込状態・ARIA 実装 | — | 良好 |
| モバイル(親指UX) | ボトムナビ中央FAB | — | 良好 |

**総評**: 「テキスト検索」としては及第だが、書籍EC・動画・医療メディアに不可欠な「視覚で一覧する(見て探す)」体験が欠落しており、現代水準から1段落ちる。ユーザー指摘は的確。

## 3. v4方針: サムネ主役への転換

### 3-1. 生成カバー(実表紙は複製しない)

実サイトの表紙画像は著作物のため取得・複製しない。代わりに、各アイテムのメタ情報(ゾーン・形式・タイトル・著者・診療科・地域・尺)から**決定論的にカバー/サムネを生成**する。

- **書籍・電子**: 縦3:4。ゾーン色のグラデ+背表紙帯+レーベル名+タイトル(明朝系serif で書籍らしさ)+著者。電子は「電子版」バッジ。
- **記事**: 4:3。診療科を大きく見せる抽象カバー(連載区分ラベル付き)。
- **動画**: 16:9。暗めサムネ+中央の再生ボタン+右下に尺(ダミー)。
- **求人・物件**: 4:3。地図ピン+地域名+種別。

### 3-2. カードのビジュアル主役化

全カード(トップ/検索結果/一覧/関連)を「カバー上・テキスト下」の構成に統一。カードは1箇所(`jmedjCard`)で生成しているため、`jmedjCover()` を差し込むだけで全画面に反映される。ホバーでカバーがわずかにズームし、影が立つ。

### 3-3. 詳細ページのビジュアル化

商品・動画の詳細メディア枠を、テキストのプレースホルダーから生成カバー(大)に置き換える。

### 3-4. アクセシビリティ配慮

カバーは装飾リンク(`aria-hidden` + `tabindex=-1`)とし、スクリーンリーダー・キーボードにはタイトルリンク/CTAの1経路のみを提示(重複リンクをa11yツリーから隠す一般的パターン)。画像が無くてもテキストで完全に成立する。

## 4. 実装対応表

| 方針 | 実装 |
|---|---|
| 生成カバー | `assets/js/app.js` の `jmedjCover(item)` |
| カード主役化 | `jmedjCard()` 改修 + `style.css` の `.cover*` / `.card-body` |
| 詳細ビジュアル | `product.js` / `video.js` が `jmedjCover(item)` を詳細枠に注入 |
| 動画の尺 | `data/sample-content.json` に `duration` 追加 |
