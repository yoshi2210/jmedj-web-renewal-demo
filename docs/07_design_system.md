# Step 6 実行結果: デザインシステム

`mock/assets/css/style.css` に実装済みのデザイントークンを正式なリファレンスとして文書化する。追加ページを実装する際は、ここに定義したトークンのみを使い、新しい色・余白単位を増やさない。

## カラー

| 用途 | トークン | 値 | 適用箇所 |
|---|---|---|---|
| 基本テキスト | `--color-ink` | `#14213d` | 本文・見出し |
| 補助テキスト | `--color-ink-soft` | `#3c4a68` | メタ情報・キャプション |
| 背景 | `--color-bg` | `#f6f7fa` | ページ背景 |
| カード面 | `--color-surface` | `#ffffff` | カード・ヘッダー |
| 境界線 | `--color-border` | `#dfe3ec` | カード枠・区切り線 |
| アクセント(CTA) | `--color-accent` | `#0f6e6a` | 主要CTA・リンク |

### ゾーンカラー(4事業を識別する唯一の視覚差分)

| ゾーン | メインカラー | 背景色(バッジ用) |
|---|---|---|
| 出版物(Zone A) | `--zone-books` `#1b3a6b` | `--zone-books-bg` `#eaf0fb` |
| 記事(Zone B) | `--zone-articles` `#1f7a4d` | `--zone-articles-bg` `#e8f6ee` |
| 動画(Zone C) | `--zone-videos` `#6a3fa0` | `--zone-videos-bg` `#f1eafb` |
| 求人・物件(Zone D) | `--zone-career` `#c1611a` | `--zone-career-bg` `#fcefe2` |

**原則**: ゾームカラーは「バッジ」「ゾーンドット」「メガメニューのボタン文字色」以外には使わない。本文・背景・CTAには使わない(ゾーン間の一貫したブランド体験を保つため、02章で定義した「共通デザイン言語の中で識別可能にする」方針)。

## スペーシングスケール(4pxベース)

| トークン | 値 | 用途 |
|---|---|---|
| `--space-1` | 4px | アイコンと文字の間隔など最小単位 |
| `--space-2` | 8px | バッジ内側余白、ボタン内側余白 |
| `--space-3` | 16px | カード内側余白、要素間の標準間隔 |
| `--space-4` | 24px | セクション内のブロック間隔 |
| `--space-5` | 40px | 空状態・ローディングの上下余白 |
| `--space-6` | 64px | セクション間の大きな区切り |

## タイポグラフィ

- フォント: `Noto Sans JP`(現行サイトと同一。刷新時の学習コスト・ブランド一貫性を優先し変更しない)
- 見出し: 30px(ヒーロー) / 20px(セクション見出し) / 16px(カード内見出し)
- 本文・メタ情報: 14px / 13px / 12px の3段階のみに制限(現行サイトはドロワー内でフォントサイズがリンクごとに不統一)

## コンポーネント

| コンポーネント | 定義箇所 | 再利用ルール |
|---|---|---|
| `.card` | 全ゾーン共通のカード | タイトル・メタ・バッジ・CTAの4要素構成を崩さない。ゾーンごとに要素を増減させない |
| `.badge` | ゾーン識別バッジ | 常にカード左上、1枚のカードにつき1つのみ |
| `.mega-panel` | メガメニュー展開パネル | 1パネルあたり見出し(h3)2つ以内、リンク8個以内(Step 4 IA設計のミラーのロー適用) |
| `.tab-btn` | ゾーン別タブ切り替え(検索結果・一覧ページ共通) | 5個以内(すべて+4ゾーン) |
| `.empty-state` / `.loading-row` | 検索・一覧の状態表示 | 新規一覧ページを追加する際は必ずこの2状態+結果状態の3点セットを実装する(08章 QA基準) |

## 適用範囲

このデザインシステムは、Step 8で追加する `product.html` / `shop-books.html` / `article.html` / `career.html` / `account.html` にもすべて同一トークン・同一コンポーネントで適用する(新規CSSの追加は既存トークンの組み合わせに限定し、`style.css`に追記する形で行い、ページ固有のインラインスタイルは持たない)。
